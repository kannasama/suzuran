# .impeccable.md

## Design Context

### Project
**suzuran** — a personal music library manager (beets alternative). Handles organization, tagging, and encoding format management. Built with Tauri + web frontend.

### Users
Primarily the developer themselves, potentially shared publicly. Power users who know their music library intimately — they care about metadata accuracy, batch operations, and format control. They expect a tool that respects their expertise and doesn't hand-hold.

### Brand Personality
**Precise. Refined. Purposeful.**

The interface should feel like a well-crafted instrument — not a consumer app, not a rough CLI wrapper. It earns trust through consistency and information density done right.

### Emotional Goals
- Confidence: the user always knows what state their library is in
- Control: data and actions are always within reach
- Calm focus: no noise, no unnecessary animation, no visual chaos

### Aesthetic Direction
**Dense-utilitarian crossed with elegant-refined** — foobar2000 DarkOne series is the reference aesthetic. Key characteristics:
- Dark-first (primary experience is dark mode, foobar2000-style dark panels)
- Dense tabular layouts for track/album lists — information is the UI
- Cool blue / indigo accent on a near-black background
- Subtle, low-contrast panel separators rather than heavy borders
- Monospace or technical sans-serif for metadata fields
- Minimal chrome: no drop shadows, no rounded corners on data tables
- Light mode exists but is secondary — same density, inverted palette

**Anti-references:** Avoid Spotify-style large artwork grids, consumer streaming UI conventions, heavy gradients, or anything that prioritizes discovery over management.

### Design Principles

1. **Data is the interface** — metadata fields, file paths, and tags are primary citizens. Typography and layout serve their legibility, not the other way around.

2. **Density with breath** — pack information like foobar2000, but use consistent spacing and alignment to prevent visual noise. One well-chosen gap is worth ten padding tweaks.

3. **Dark-first, light-parity** — design for dark mode first. Light mode must achieve the same density and hierarchy, not a washed-out inversion.

4. **Precision over decoration** — every visual element must earn its place. No gradients, illustrations, or animations that don't communicate state or aid comprehension.

5. **Power-user affordances** — keyboard navigation, multi-select, bulk operations, and status feedback are first-class. The interface assumes the user knows what they're doing.

### Color Palette Direction
- Background: near-black (`#1a1a1a` range), with slightly lighter panel surfaces
- Accent: cool blue / indigo (e.g., `#4f8ef7` or similar)
- Text: high-contrast white primary, muted gray for secondary metadata
- Destructive: desaturated red, not alarming
- Success/encoding states: muted green

### Typography Direction
- UI labels and headings: clean geometric sans-serif (Inter, Geist, or similar)
- Metadata / tag values / file paths: monospace or tabular-figures sans for alignment
- Small sizes acceptable — users are comfortable with dense technical UIs
